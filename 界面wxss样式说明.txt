当前项目界面 wxss 样式说明

一、整体设计方向
1. 当前 home、user、statis 三个页面统一采用浅绿色到浅灰绿色的纵向渐变背景。
2. 核心卡片统一使用大圆角、白底或绿色渐变底、轻阴影，形成“小程序宠物健康管理”风格。
3. 数值单位统一使用 rpx，这样在不同手机尺寸下缩放更自然。

二、pages/home/home.wxss 说明
1. .home-page
   min-height: 100vh
   作用：保证页面至少铺满一整屏。
   效果：即使内容不多，也不会露出底色空白断层。

   padding: 32rpx 24rpx 40rpx
   作用：给页面四周留出安全边距。
   效果：卡片不会紧贴屏幕边缘，视觉更舒展。

   background: linear-gradient(180deg, #eef8f4 0%, #f7fbfa 45%, #f4f7f8 100%)
   作用：从上到下做浅绿色渐变。
   效果：页面看起来更柔和，不会像纯白底那样生硬。

2. .hero-card
   padding: 36rpx 30rpx
   作用：控制顶部宠物卡片内部留白。
   效果：文字、头像、状态不会挤在一起。

   border-radius: 32rpx
   作用：大圆角。
   效果：符合当前页面“柔和、健康管理类”的视觉方向。

   background: linear-gradient(135deg, #1f5f5b 0%, #3b8b7f 100%)
   作用：绿色渐变主卡片。
   效果：让顶部卡片成为首页最重要的视觉焦点。

   box-shadow: 0 18rpx 36rpx rgba(31, 95, 91, 0.18)
   作用：添加柔和阴影。
   效果：卡片有悬浮感，不会贴在背景上。

3. .hero-title
   font-size: 48rpx
   作用：突出宠物名字。
   效果：进入页面时最先看到宠物主体信息。

4. .hero-profile 与 .hero-profile-item
   display: flex + flex-wrap: wrap
   作用：将品种、性别、年龄、体重做成两列排列。
   效果：比单行平铺更整齐，也比一整块说明文字更易读。

   .hero-profile-item width: 50%
   作用：每行两个字段。
   效果：顶部卡片空间利用更高，不需要单独再开新卡片。

5. .hero-status
   padding: 10rpx 18rpx
   border-radius: 999rpx
   background: rgba(255, 255, 255, 0.16)
   作用：做成胶囊状态条。
   效果：状态信息独立于档案信息，层级更清楚。

6. .hero-image
   width: 176rpx
   height: 176rpx
   border-radius: 28rpx
   作用：固定宠物头像大小。
   效果：头像稳定清晰，不会随着文字变化被挤压。

7. .metric-card / .overview-card / .module-card
   border-radius: 28rpx
   background: #ffffff
   box-shadow: 0 12rpx 28rpx rgba(25, 46, 44, 0.08)
   作用：统一内容卡片视觉语言。
   效果：整个首页不会出现风格割裂。

三、pages/user/user.wxss 说明
1. .user-page
   background: linear-gradient(180deg, #eef7f4 0%, #f7fbfa 52%, #f2f6f5 100%)
   作用：与首页保持相同色系，但稍微更淡。
   效果：编辑页更安静，减少对操作的干扰。

2. .editor-card
   padding: 28rpx
   border-radius: 32rpx
   background: #ffffff
   box-shadow: 0 14rpx 32rpx rgba(27, 54, 50, 0.08)
   作用：形成主要编辑容器。
   效果：用户一眼就知道当前操作区域。

3. .avatar-area
   display: flex
   align-items: center
   border-bottom: 1rpx solid #edf1f0
   作用：头像区域左右布局，并与表单区域做分隔。
   效果：结构清晰，不会混成一整块。

4. .avatar-image
   width: 124rpx
   height: 124rpx
   border-radius: 28rpx
   作用：头像统一尺寸。
   效果：头像既足够清晰，又不会抢过输入区域的视觉重点。

5. .field-input
   height: 88rpx
   line-height: 88rpx
   padding: 0 24rpx
   border-radius: 22rpx
   background: #f5f8f7
   作用：固定输入框高度和文字垂直居中。
   效果：解决之前输入文字显示不全、文字被截断的问题。

6. .action-row
   display: flex
   justify-content: space-between
   作用：让“恢复默认”和“保存修改”并排显示。
   效果：主要操作集中，用户不用上下找按钮。

7. .reset-btn / .save-btn / .logout-btn
   display: flex
   align-items: center
   justify-content: center
   作用：让按钮文字真正垂直水平居中。
   效果：修复了按钮文字偏下的问题。

   border-radius: 999rpx
   作用：全部使用胶囊按钮。
   效果：和页面圆角卡片语言一致，更柔和。

8. .reset-btn
   background: #edf4f2
   color: #1f5f5b
   作用：次级按钮。
   效果：让“恢复默认”不会抢掉“保存修改”的主操作地位。

9. .save-btn
   background: linear-gradient(135deg, #1f5f5b 0%, #3a8d80 100%)
   color: #ffffff
   作用：主操作按钮。
   效果：用户能快速识别“保存修改”是最重要操作。

10. .avatar-btn::after 等去边框
    border: none
    作用：去掉微信 button 默认边框。
    效果：按钮更干净，也不会影响居中视觉。

四、pages/statis/statis.wxss 说明
1. .statis-page

2. .hero-card

6. .chart-head

    效果：异常点详情更醒目，但不会像纯红色那样太刺激。

五、为什么这些数值能形成当前效果
1. 24rpx、28rpx、32rpx 这几个值大量用于 padding 和圆角。
   这样卡片之间会形成统一的圆润感，是现在页面统一风格的关键。

2. 88rpx 按钮和输入框高度是为了适配手机触控。
   高度足够，点按舒服，同时文字也容易做垂直居中。

3. 阴影透明度都控制在 0.06 到 0.18 之间。
   这样能看出层级，但不会像重阴影那样显得廉价或压迫。

4. 标题字号一般在 30rpx 到 48rpx。
   大标题负责建立主次，小标题负责说明，不会互相抢视觉。

5. 渐变色统一使用绿色系。
   这是整个“宠物健康监测系统”的主视觉线索，能让三页看起来属于同一项目。

六、如果后续想继续调整，通常改哪里
1. 想让页面更清爽：
   减小阴影，减少说明文字，增加留白。

2. 想让页面更科技感：
   提高主卡片颜色对比，图表线条颜色更亮，背景更干净。

3. 想让页面更温和：
   继续降低绿色饱和度，卡片圆角略增，文字颜色更浅。

4. 想让图表更醒目：
   增加 ec-canvas 高度，减少同屏卡片数量，放大图表标题和异常点颜色对比。
